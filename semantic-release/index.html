Versioning

Important resource to read: https://semver.org/ to understand the versioning. For .NET it's recommended to read https://github.com/Microsoft/api-guidelines/blob/master/Guidelines.md

Eeach project team should decide by themselves how to use the versioning for backend, frontend and mobile.

NET Versioning

https://github.com/Microsoft/api-guidelines/blob/master/Guidelines.md This document explains how to use the versioning for .NET projects. https://dateo-software.de/blog/web-api-versioning

Project specifications

C# backend projects

In .csproj this code should be defined in 

  <VersionPrefix>1.0.0</VersionPrefix>
  <VersionSuffix>$([System.DateTime]::UtcNow.ToString(`yyyyMMdd-HHmm`))</VersionSuffix>
```

Manually versioning Angular

In environments add

version: '1.0.0',

Add in .ts file:

  version: string = '1.0';
  year = 0;
  custom = {
    version: true,
  };

  constructor() {}

  ngOnInit(): void {
    this.version = environment.version;
    this.year = new Date().getFullYear();
  }

Add in html file:

<p class="mb-1">Â© {{ year }} AppName v.{{ version }}</p>
